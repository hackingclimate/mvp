This means that unless you want to change the architecture of the site (adding/removing sections, reorganizing, etc) you should be able to just create and edit Markdown docs.

Examples ğŸ‘€

If you want an example of what Docusaurus can do, look no further than the official Docusaurus website. Another great example is hack.aragon. Since Docusaurus is based on React, you can extend and customize it as little or as much as you'd like.

Docs ğŸ“š

Docusaurus has great docs. Rather than recreating them, I'll just link to the most useful ones for editing this site.

  • Navigation and Sidebars - this explains how to link a new Markdown document to the rest of the webiste.

  • Publishing your site - TL;DR you have to uncomment some stuff in siteConfig.js, then navigate to the website directory, then run the script below

    GIT_USER=your-username
    CURRENT_BRANCH=master
    yarn run publish-gh-pages

When you're done publishing, remember to go back to siteConfig.js and uncomment the publish params and go back to the local dev build!
